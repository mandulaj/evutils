:py:mod:`evutils.transforms`
============================

.. py:module:: evutils.transforms

.. autoapi-nested-parse::

   Data augmentation transforms for events.

   torchvision/tonic-style transformations for event streams: random and
   deterministic operations for augmenting data during training (drops, spatial
   flips, jitter, time skew, refractory filtering, ...).

   Each transform pairs a Numba-JIT ``functional`` kernel (dispatched over both
   plain structured arrays and :class:`~evutils.types.EventArray`) with a
   composable :class:`Transform` class. :class:`Compose` unwraps/rewraps events
   only once around each contiguous block of JIT transforms.

.. py:class:: Compose(transforms: Iterable[Callable])

   Composes several transforms together.

   Groups contiguous blocks of evutils `Transform` objects to execute them purely in
   C-space via Numba JIT without intermediate unpacking/repacking overhead.
   Freely accepts standard Callables (e.g. PyTorch/Tonic transforms).

.. py:class:: Transform

   Base class for all evutils transforms.

   Transforms should implement the `_forward_jit` method to allow zero-overhead
   composition inside a `Compose` pipeline.


   .. py:method:: bind_context(events)

      Capture per-call context (currently ``sensor_size``) from ``events``.

      Called by :meth:`__call__` and by :class:`Compose` before ``_forward_jit``
      so a transform can fall back to the container's ``sensor_size`` metadata
      when it was not given one explicitly. Stored transiently, not persisted.

   Randomly drops each event with probability ``p``.

   Tonic-compatible replacement for ``DropRandomEvents`` (kept as an alias).
   Uses an independent Bernoulli mask per event, so the number dropped is
   binomially distributed around ``p * n`` rather than exactly ``p * n``.

   :param p: Drop probability in ``[0, 1)``. A ``(lo, hi)`` tuple is sampled uniformly
             per call. Defaults to ``0.1``.
   :type p: float or tuple of float, optional

.. py:class:: TimeNormalize(start_ts: int = 0)

   Bases: :py:obj:`Transform`

   .. autoapi-inheritance-diagram:: evutils.transforms.TimeNormalize
      :parts: 1


   Shifts timestamps so the earliest event lands at ``start_ts``.

   Stateless: each call normalizes the batch it is given on its own. For a
   chunk-by-chunk stream use ``EventReader(normalize_ts=True)``, which latches
   the offset from the first chunk across the whole stream; dropping this
   transform into a ``Compose`` over sequential chunks would instead reset each
   chunk to ``start_ts`` independently.

   :param start_ts: Timestamp assigned to the earliest event. Default ``0``.
   :type start_ts: int, optional
