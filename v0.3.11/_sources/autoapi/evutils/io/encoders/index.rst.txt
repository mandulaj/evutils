:py:mod:`evutils.io.encoders`
=============================

.. py:module:: evutils.io.encoders

.. autoapi-nested-parse::

   Encoders module.

   Provides mapping and retrieval of event encoders based on file extensions.
   Backends whose optional dependencies are missing (e.g. pandas for CSV, h5py
   for HDF5) are not registered; asking for them raises an ``ImportError`` that
   names the extra to install.

      :parts: 1


   Encode events into an ``.npz`` archive.

   Chunks passed to :meth:`write` are buffered in memory; the archive itself
   is written once on :meth:`close` (zip containers cannot be appended to).

   :param writable: Destination stream to write to.
   :param width: Frame geometry stored in the archive.
   :type width: int
   :param height: Frame geometry stored in the archive.
   :type height: int
   :param dt: Unused; npz stores no recording timestamp.
   :type dt: datetime, optional
   :param compressed: Use ``np.savez_compressed`` instead of ``np.savez``.
   :type compressed: bool


   .. py:method:: init() -> None

      Nothing to write up front; the archive is produced on close.



   .. py:method:: write(events: numpy.ndarray | EventArray) -> int

      Buffer a chunk of events for the archive.

      :param events: Array of events to write.
      :type events: np.ndarray or EventArray

      :returns: Number of buffered events.
      :rtype: int



   .. py:method:: flush() -> None

      No-op: the archive can only be written once, on :meth:`close`.



   .. py:method:: close() -> None

      Concatenate the buffered chunks and write the archive.

.. py:class:: EventEncoder_AER(writable, width: int = 512, height: int = 512, dt=None)

   Bases: :py:obj:`evutils.io.common.EventEncoder`

   .. autoapi-inheritance-diagram:: evutils.io.encoders.EventEncoder_AER
      :parts: 1


   Encode events into a raw AER stream. Since AER is designed for real-time
   streaming, it has no header and no timestamps.
   Timestamps are dropped and coordinates are masked to 9 bits (values >= 512
   are truncated), per the AER encoding.

   :param writable: Destination stream to write to.
   :param width: Frame geometry written into the header.
   :type width: int
   :param height: Frame geometry written into the header.
   :type height: int
   :param dt: No effect, since AER has no timestamps.
   :type dt: datetime, optional

   .. rubric:: References

   [1] Prophesee AER format: https://docs.prophesee.ai/stable/data/encoding_formats/aer.html
