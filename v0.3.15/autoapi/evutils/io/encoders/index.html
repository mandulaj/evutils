evutils.io.encoders#

Encoders module.

Provides mapping and retrieval of event encoders based on file extensions. Backends whose optional dependencies are missing (e.g. pandas for CSV, h5py for HDF5) are not registered; asking for them raises an ImportError that names the extra to install.

Encode events into an .npz archive with bounded memory.

Zip members can only be written one after another, while write() receives all four columns interleaved – so each column is spooled to an unlinked temporary file (raw bytes, no size limit from RAM) and the archive is assembled on close() by streaming every spool into its .npy member.

Parameters:

  • writable (io.BufferedIOBase) – Destination stream to write to.

  • width (int) – Frame geometry stored in the archive.

  • height (int) – Frame geometry stored in the archive.

  • dt (datetime, optional) – Unused; npz stores no recording timestamp.

  • compressed (bool) – Deflate the archive members (like np.savez_compressed).

Encode events into a raw AER stream. Since AER is designed for real-time streaming, it has no header and no timestamps. Timestamps are dropped and coordinates are masked to 9 bits (values >= 512 are truncated), per the AER encoding.

Parameters:

  • writable (io.BufferedWriter) – Destination stream to write to.

  • width (int) – Frame geometry written into the header.

  • height (int) – Frame geometry written into the header.

  • dt (datetime, optional) – No effect, since AER has no timestamps.

References
