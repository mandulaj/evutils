.. py:function:: drop_event(events, p=0.1)

   Randomly drop each event independently with probability ``p``.

   The tonic-compatible name for :func:`drop_random_events`. Unlike tonic's
   ``drop_event_numpy`` (which drops exactly ``round(p * n)`` events via
   sampling without replacement) this uses an independent Bernoulli mask per
   event, which is what keeps the kernel JIT-friendly. The expected number of
   dropped events is the same; the exact count is binomially distributed.

   :param events: Events to drop from.
   :type events: np.ndarray or EventArray
   :param p: Drop probability in ``[0, 1)``. A ``(lo, hi)`` tuple is sampled uniformly
             per call. Defaults to ``0.1``.
   :type p: float or tuple of float, optional

   :returns: Events that survived the drop, in their original container type.
   :rtype: np.ndarray or EventArray


.. py:function:: drop_by_time(events, duration_ratio=0.2)

   Drop every event inside one randomly-placed time window.

   The window length is ``duration_ratio`` of the recording span (``[0, t.max()]``,
   following tonic), positioned uniformly at random within it.

   :param events: Events to drop from.
   :type events: np.ndarray or EventArray
   :param duration_ratio: Window length as a fraction of the span, in ``[0, 1)``. A ``(lo, hi)``
                          tuple is sampled uniformly per call. Defaults to ``0.2``.
   :type duration_ratio: float or tuple of float, optional

   :returns: Events outside the dropped window, in their original container type.
   :rtype: np.ndarray or EventArray

.. py:function:: spatial_jitter(events, sensor_size, var_x=1.0, var_y=1.0, sigma_xy=0.0, clip_outliers=False)

   Jitter event coordinates with a 2D Gaussian.

   :param events: Events to jitter.
   :type events: np.ndarray or EventArray
   :param sensor_size: ``(W, H, P)`` sensor size, used for clipping.
   :type sensor_size: tuple
   :param var_x: Variances of the jitter in x and y. Default ``1.0``.
   :type var_x: float, optional
   :param var_y: Variances of the jitter in x and y. Default ``1.0``.

   :type sigma_xy: float, optional
   :param clip_outliers: Drop events jittered outside the sensor instead of casting them back

   :type clip_outliers: bool, optional

   :rtype: np.ndarray or EventArray

.. py:function:: normalize_ts(events, start_ts=0)

   Shift timestamps so the earliest event lands at ``start_ts``.

   Pure and stateless: each call normalizes the batch it is handed on its own.
   For chunk-by-chunk streams use ``EventReader(normalize_ts=True)``, which
   latches the offset from the first chunk and applies it across the whole
   stream (so the timeline stays continuous instead of resetting per chunk).

   :param events: Events to normalize.
   :type events: np.ndarray or EventArray
   :param start_ts: Timestamp assigned to the earliest event. Default ``0``.
   :type start_ts: int, optional

   :returns: Events with shifted timestamps, in their original container type. The
             input is not modified.
   :rtype: np.ndarray or EventArray

   .. rubric:: Examples

   >>> import numpy as np
   >>> from evutils.transforms.functional import normalize_ts
   >>> events = np.array(
   ...     [(0, 0, 100, 1), (1, 1, 200, 1), (2, 2, 300, 0)],
   ...     dtype=[('x', 'u2'), ('y', 'u2'), ('t', 'i8'), ('p', 'i1')]
   ... )
   >>> normalize_ts(events)['t']
   array([  0, 100, 200])


.. py:function:: refractory_period(events, delta)

   Enforce a per-pixel refractory period.

   :param events: Events to filter. Must be sorted by timestamp.
   :type events: np.ndarray or EventArray
   :param delta: Refractory period in the same time unit as the timestamps. Events at a
                 pixel that fire within ``delta`` of that pixel's previous event are
                 dropped.
   :type delta: int

   :returns: Filtered events, in their original container type.
   :rtype: np.ndarray or EventArray
