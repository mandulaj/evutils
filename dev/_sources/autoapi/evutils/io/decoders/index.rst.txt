:py:mod:`evutils.io.decoders`
=============================

.. py:module:: evutils.io.decoders

.. autoapi-nested-parse::

   Decoders module.

   Provides mapping and resolution of event decoders based on file extensions or
   magic bytes. Backends whose optional dependencies are missing (e.g. pandas for
   CSV, h5py for HDF5) are not registered; asking for them raises an
   ``ImportError`` that names the extra to install.

      Seek to an absolute timestamp (µs) or event index. See base class.

      Fixed 8-byte records: event ``k`` lives at word offset ``2k`` and its
      timestamp is the even-strided word ``_words[2k]``. Time seek is a

      raw 32-bit timestamps are non-decreasing (no wrap).

   Decode events from an ``.npz`` archive, streaming chunk by chunk.

   The archive members are read through zip streams: only ``chunk_size``
   events are held in memory at a time, so arbitrarily large recordings can
   be iterated. Accepts either the four column members ``t/x/y/p`` or a
   single structured ``events`` member.

   :param source: Byte source to read from (must be seekable, as required by the zip
                  format).
   :param chunk_size: Number of events returned per :meth:`read_chunk` call.


   .. py:attribute:: SUPPORTS_SEEK

   .. py:method:: parse_step_delta_t(events: evutils.io._native_core.EventSoABuffers, triggers: evutils.io._native_core.TriggerSoABuffers, end_ts: int) -> tuple[int, int]

      Run the dedicated C delta_t parser once, appending events with
      timestamp ``< end_ts`` into ``events``.

      Returns ``(appended, status)`` where status is one of
      ``EVUTILS_PARSE_WINDOW_DONE`` (the time boundary was reached -- the
      window is complete), ``EVUTILS_PARSE_OUTPUT_FULL`` (grow and call again),
      or ``EVUTILS_PARSE_OK`` (input drained for this call). At true EOF the
      final incomplete vector group is flushed via the tail pad, mirroring
      :meth:`parse_step`.

.. py:class:: EventDecoder_AER(source: evutils.io._source.ByteSource, chunk_size: int = 1000000, timestamps: str | np.ndarray = 'zero', t_start: int = 0, t_step: int = 1)

   Decode raw AER streams into ``EventArray`` chunks. Since AER is designed
   for real-time streaming, it has no header and no timestamps; see the
   ``timestamps`` parameter for how the ``t`` column is generated.

   :param source: Byte source to read from.
   :param chunk_size: Maximum number of events produced per :meth:`read_chunk` call (the
                      native output-buffer capacity). Does not bound the file size.
   :param timestamps: Timestamp generation mode: ``"zero"`` fills ``t = 0``,
                      ``"sequential"`` fills ``t = t_start + i * t_step``, and an integer
                      array assigns user-provided timestamps positionally (its length must
                      cover every decoded event).
   :type timestamps: {"zero", "sequential"} or array_like, default "zero"
   :param t_start: Start value and per-event increment for ``"sequential"`` mode.
   :type t_start: int
   :param t_step: Start value and per-event increment for ``"sequential"`` mode.
   :type t_step: int

   .. rubric:: References

   [1] Prophesee AER format: https://docs.prophesee.ai/stable/data/encoding_formats/aer.html

.. py:function:: resolve_decoder_cls(source: io.BufferedIOBase | str | bytes) -> Type[evutils.io.common.EventDecoder]

   Determine the decoder class for a :class:`ByteSource`.

   Tries the filename extension first (cheap, usually right), then falls back
   to sniffing the leading bytes -- which works for streams and USB devices
   that have no filename.

   :param source: A ByteSource (see :mod:`evutils.io._source`).

   :returns: The decoder class to instantiate with the source.
   :rtype: Type[EventDecoder]
