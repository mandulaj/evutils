evutils.transforms.functional.drop_event(events, p=0.1, seed: int | None = None)[source]#

Randomly drop each event independently with probability p.

The tonic-compatible name for drop_random_events(). Unlike tonic’s drop_event_numpy (which drops exactly round(p * n) events via shuffle), this drops independently, which is O(N) rather than O(N log N). (sampling without replacement) this uses an independent Bernoulli mask per event, which is what keeps the kernel JIT-friendly. The expected number of dropped events is the same; the exact count is binomially distributed.

Parameters:

  • events (np.ndarray or EventArray) – Events to drop from.

  • p (float or tuple of float, optional) – Drop probability in [0, 1). A (lo, hi) tuple is sampled uniformly per call. Defaults to 0.1.

  • seed (int | None)

Returns:

Events that survived the drop, in their original container type.

Return type:

np.ndarray or EventArray

evutils.transforms.functional.drop_by_time(events, duration_ratio=0.2, seed: int | None = None)[source]#

Drop every event inside one randomly-placed time window.

The window length is duration_ratio of the recording span ([t.min(), t.max()], following tonic), positioned uniformly at random within it.

Parameters:

  • events (np.ndarray or EventArray) – Events to drop from.

  • duration_ratio (float or tuple of float, optional) – Window length as a fraction of the span, in [0, 1). A (lo, hi) tuple is sampled uniformly per call. Defaults to 0.2.

  • seed (int, optional) – Seed for the random number generator. Defaults to None.

Returns:

Events outside the dropped window, in their original container type.

Return type:

np.ndarray or EventArray

evutils.transforms.functional.normalize_ts(events, start_ts=0)[source]#

Shift timestamps so the earliest event lands at start_ts.

Pure and stateless: each call normalizes the batch it is handed on its own. For chunk-by-chunk streams use EventReader(normalize_ts=True), which latches the offset from the first chunk and applies it across the whole stream (so the timeline stays continuous instead of resetting per chunk).

Parameters:

  • events (np.ndarray or EventArray) – Events to normalize.

  • start_ts (int, optional) – Timestamp assigned to the earliest event. Default 0.

Returns:

Events with shifted timestamps, in their original container type. The input is not modified.

Return type:

np.ndarray or EventArray

Examples

>>> import numpy as np
>>> from evutils.transforms.functional import normalize_ts
>>> events = np.array(
...     [(0, 0, 100, 1), (1, 1, 200, 1), (2, 2, 300, 0)],
...     dtype=[('x', 'u2'), ('y', 'u2'), ('t', 'i8'), ('p', 'i1')]
... )
>>> normalize_ts(events)['t']
array([  0, 100, 200])
